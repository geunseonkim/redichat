## 21. 앱 안정성 재개선 (종료 및 입력 오류 수정)

- **입력 오류 및 종료 에러 재수정**: `useInput` 훅에서 비동기 클린업을 직접 처리할 때 발생하는 렌더링 오류와 `Connection is closed` 에러를 수정했습니다.
- **원인**: `useInput` 핸들러 내에서 `async` 함수를 실행하고 바로 `exit()`를 호출하려 할 때, 비동기 작업이 완료되기 전에 프로세스가 종료되려 하거나, Ink의 렌더링 루프와 충돌이 발생했습니다.
- **해결**: `isExiting` 상태를 도입하여 종료 프로세스를 분리했습니다. `useInput`은 `isExiting` 상태만 변경하고, 별도의 `useEffect` 훅이 이 상태 변화를 감지하여 안전한 비동기 클린업 작업을 수행한 후 `exit()`를 호출하도록 구조를 변경했습니다. 이를 통해 안정적인 종료와 정상적인 렌더링을 보장합니다.

## 22. 배포 전략 수립

- 애플리케이션 배포 전략을 수립했습니다. 배포는 **Redis 서버**와 **클라이언트 앱** 두 부분으로 나뉩니다.
- **Redis 서버**: 모든 사용자가 접속할 수 있도록 인터넷에 공개된 서버에 호스팅해야 합니다.
  - **권장 방식**: Redis Cloud, AWS ElastiCache 같은 관리형(Managed) 서비스를 사용합니다.
  - **주의**: Vercel, Netlify와 같은 정적/서버리스 플랫폼은 Redis 호스팅에 적합하지 않음을 명확히 했습니다.
- **클라이언트 앱**: 각 사용자의 터미널에서 실행되는 앱입니다.
  - **권장 방식**: `pkg`와 같은 도구를 사용하여 OS별(macOS, Windows, Linux) 독립 실행 파일로 패키징합니다.
  - **배포처**: GitHub Releases를 통해 패키징된 파일을 배포하여 사용자들이 쉽게 다운로드할 수 있도록 합니다.
- 관련 내용을 `docs/architecture.md` 문서에 상세히 기록했습니다.

## 23. 날짜 변경 시 날짜 구분선 표시 기능 추가

- 채팅 기록에서 날짜가 바뀔 때마다, `--- YYYY년 M월 D일 ---` 형식의 날짜 구분선이 표시되도록 UI를 개선했습니다.
- 메시지 목록을 렌더링할 때, 이전 메시지와 현재 메시지의 날짜를 비교하여 구분선 표시 여부를 동적으로 결정합니다.
- 이를 통해 사용자는 시간의 흐름을 더 명확하게 파악할 수 있습니다.

## 24. 채팅방 이동 기능 추가 (/join)

- 앱을 재시작하지 않고 다른 채팅방으로 이동할 수 있는 `/join` 및 `/join <방이름>` 명령어를 추가했습니다.
- 현재 방을 나가는 로직(LEAVE 메시지 발행, 사용자 목록에서 제거)을 실행한 후, 새로운 방에 입장하는 흐름으로 연결됩니다.
- `/join <방이름>`을 사용하면 바로 해당 방의 입장 절차(비밀번호 입력 등)를 밟게 됩니다.
- `useEffect` 훅의 정리(cleanup) 함수에 `subscriber.unsubscribe()`를 추가하여, 방을 이동할 때 이전 채널의 구독이 확실하게 해제되도록
  수정했습니다.

## 25. 앱 안정성 최종 개선 (한글 입력 및 종료 오류)

- **한글 입력 오류 분석**: 한글 입력 시 글자가 터미널 하단에 표시되는 현상은 `ink-text-input` 라이브러리가 멀티바이트 문자 조합을 처리하는 과정에서 발생하는 알려진 한계점임을 확인했습니다. 이는 시각적인 불편함은 있지만, 앱의 핵심 기능에는 영향을 주지 않습니다.
- **종료 오류 수정**: `Ctrl+C`로 종료 시 `Error: Connection is closed.` 에러가 발생하는 문제를 해결했습니다.
  - **원인**: `ioredis`의 `disconnect()` 함수가 연결을 즉시 끊어버려, 아직 처리 중이던 `publish` 같은 명령이 실패하면서 에러가 발생했습니다.
  - **해결**: `disconnect()` 대신, 대기 중인 모든 명령을 처리한 후 안전하게 연결을 종료하는 `quit()` 함수를 사용하도록 변경하여, 에러 없이 깔끔하게 종료되도록 수정했습니다.
