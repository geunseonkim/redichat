# Contributing to Redis Terminal Chat

First off, thank you for considering contributing! It's people like you that make open source such a great community.

## How Can I Contribute?

### Reporting Bugs

- Ensure the bug was not already reported by searching on GitHub under [Issues](https://github.com/your-username/redis-chat/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/your-username/redis-chat/issues/new). Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample or an executable test case** demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- Open a new issue to discuss your idea. This is a great way to get feedback before you start working on it.

### Pull Requests

1.  Fork the repository and create your branch from `main`.
2.  Set up your development environment (`npm install`).
3.  Make your changes. Please adhere to the existing code style. We use `xo` for linting.
4.  Ensure your code lints (`npm run lint` - you may need to add this script to `package.json`).
5.  Issue that pull request!

## Code of Conduct

This project and everyone participating in it is governed by the Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior.

## Styleguides

We use `xo` to enforce a consistent code style. Before committing, please run the linter to check for any issues.

```json
// package.json scripts
"lint": "xo"
```
