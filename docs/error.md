# 에러 트러블슈팅 기록 (Error Troubleshooting Log)

이 문서는 프로젝트 개발 중 발생했던 주요 에러와 그 해결 과정을 기록합니다.

## 1. 한글 입력 시 렌더링 오류

- **증상**: 닉네임이나 메시지를 입력할 때, 한글을 입력하면 글자가 터미널 화면 맨 아래에 표시되거나, 이전에 렌더링된 화면 일부가 깨져 보이는 현상. 영어 입력 시에는 문제가 발생하지 않음.
- **원인 분석**:
  - 이 문제는 `ink-text-input` 라이브러리가 터미널 환경에서 한글과 같은 조합형 문자(멀티바이트 문자)를 처리하는 방식의 한계로 인해 발생합니다.
  - '한'이라는 글자를 입력하기 위해 'ㅎ', 'ㅏ', 'ㄴ'을 순서대로 누르는 조합 과정에서, 라이브러리가 커서의 위치를 정확하게 계산하지 못하고 렌더링 오류를 일으킵니다.
- **결론**:
  - 이는 라이브러리 자체의 한계점이므로, 애플리케이션 코드 수정만으로는 근본적인 해결이 어렵습니다.
  - 다행히 입력된 값 자체는 정상적으로 처리되므로, **시각적인 불편함** 외에 앱의 핵심 기능에는 영향을 주지 않습니다.

## 2. 앱 종료(`Ctrl+C`) 관련 문제

### 2.1. `Error: Connection is closed.` 에러 발생

- **증상**: `Ctrl+C`로 앱을 종료할 때, `ioredis`에서 `Connection is closed` 에러가 발생하며 비정상적으로 종료됨.
- **원인 분석**:
  - 앱 종료 시, "채팅방을 나갔습니다" 라는 마지막 메시지를 `publish`하는 비동기 작업이 완료되기 전에, `ioredis`의 `disconnect()` 함수가 Redis 연결을 강제로 먼저 끊어버리는 **경합 조건(Race Condition)**이 발생했습니다.
  - 이미 닫힌 연결에 `publish` 명령을 보내려다 에러가 발생한 것입니다.
- **해결**:
  - 연결을 즉시 끊는 `disconnect()` 대신, `quit()` 함수를 사용하도록 변경했습니다.
  - `quit()` 함수는 대기 중인 모든 명령(예: `publish`)이 처리될 때까지 기다린 후, 안전하게(gracefully) 연결을 종료하므로 에러가 발생하지 않습니다.

### 2.2. `Ctrl+C`를 두 번 눌러야 종료되거나, 입력 오류가 발생하는 문제

- **증상**: `Ctrl+C`를 두 번 눌러야 앱이 종료되거나, 텍스트 입력 시 커서 위치가 꼬이는 현상.
- **원인 분석**:
  - Node.js의 `process.on('SIGINT', ...)`를 직접 사용한 것이 Ink 라이브러리의 터미널 제어 방식과 충돌을 일으켰습니다.
  - 또한, Ink의 `useInput` 훅 안에서 직접 `async` 함수를 호출하여 비동기 작업을 처리하려고 시도한 것도 Ink의 렌더링 루프를 방해하여 동일한 문제를 유발했습니다.
- **해결**:
  - **역할 분리**: `Ctrl+C` 입력 감지와 실제 비동기 종료 처리를 분리했습니다.
  1.  `useInput` 훅은 `Ctrl+C`가 눌리면 `isExiting`이라는 상태 플래그만 `true`로 변경합니다. (동기적 처리)
  2.  별도의 `useEffect` 훅이 `isExiting` 상태 변화를 감지합니다.
  3.  `useEffect` 훅 안에서 안전하게 비동기 정리 작업(사용자 목록에서 제거, 퇴장 메시지 전송 등)을 수행합니다.
  4.  모든 비동기 작업이 완료된 후, `exit()` 함수를 호출하여 앱을 최종적으로 종료합니다.
  - 이 구조는 Ink의 렌더링 루프를 방해하지 않으므로, 안정적인 종료와 정상적인 텍스트 입력을 모두 보장합니다.
